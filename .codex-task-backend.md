# Codex Task: Foundry Backend Foundation

Build the backend foundation for Foundry — a FastAPI application that boots, serves a health endpoint, manages config, initializes the SQLite database with the full schema, and provides the CLI entry point.

## What Already Exists
- Repo skeleton at ~/foundry with directory structure
- backend/pyproject.toml with dependencies
- backend/foundry/default_config.toml with full config template
- Empty __init__.py files in all packages

## What To Build

### 1. backend/foundry/config.py — Configuration Management
- Use pydantic-settings to load config from:
  1. Hardcoded defaults
  2. ~/.foundry/config.toml (if exists)
  3. Environment variables (FOUNDRY_* prefix)
- Define typed settings classes matching default_config.toml structure:
  - ServerSettings(host="127.0.0.1", port=8120)
  - AuthSettings(method="none", pin="", session_timeout="24h")
  - StorageSettings(data_dir="~/.foundry")
  - AgentSettings with nested providers, interaction, confirmations
  - IngestionSettings(max_concurrent=3, transcript_method="auto")
  - ExecutionSettings(workspace_scope="project", default_timeout=600, max_concurrent=3)
  - WorkspaceSettings(auto_commit=True, reconcile_mode="detect")
  - SearchSettings(engine="fts5")
  - UISettings(theme="dark", default_layout="three-panel")
  - BackupSettings(enabled=True, interval_hours=24, keep_count=7)
  - LoggingSettings(level="INFO")
- Expand ~ in paths
- Create data_dir if it doesn't exist (with 0o700 permissions)

### 2. backend/foundry/storage/database.py — Database Layer
- Async SQLite connection using aiosqlite
- WAL mode enabled on connect
- Migration runner that finds and runs migration files in order
- Connection pool / singleton pattern

### 3. backend/foundry/storage/migrations/001_initial.py — Full Schema
Create ALL tables from the design:

```sql
projects, subprojects, resources, extraction_results, tasks, notes,
file_assets, environments, agent_sessions, agent_messages,
provenance_links, git_configs, jobs, search_index (FTS5)
```

Plus all indexes. Use the exact schema from the design doc:
- projects: id TEXT PK, name, description, status DEFAULT 'active', workspace_path, settings JSON, created_at, updated_at
- subprojects: id TEXT PK, project_id FK CASCADE, name, description, type, status DEFAULT 'discovered', workspace_path, dependencies JSON, setup_steps JSON, complexity, sort_order, created_at, updated_at
- resources: id TEXT PK, project_id FK SET NULL, type, url, title, original_filename, mime_type, metadata JSON, content_hash, raw_content_path, pipeline_status DEFAULT 'pending', pipeline_error, created_at, updated_at
- extraction_results: id TEXT PK, resource_id FK CASCADE, summary, key_concepts JSON, entities JSON, content_sections JSON, discovered_projects JSON, open_questions JSON, follow_up_suggestions JSON, model_used, token_usage JSON, raw_response, created_at
- tasks: id TEXT PK, subproject_id FK CASCADE, title, description, status DEFAULT 'todo', priority, source DEFAULT 'extracted', sort_order, created_at, updated_at
- notes: id TEXT PK, project_id FK CASCADE, subproject_id FK CASCADE, title, content, source DEFAULT 'user', created_at, updated_at
- file_assets: id TEXT PK, subproject_id FK CASCADE, path, type, description, size_bytes, created_at
- environments: id TEXT PK, subproject_id FK CASCADE, type, status DEFAULT 'planned', config JSON, path, error, created_at, updated_at
- agent_sessions: id TEXT PK, project_id FK SET NULL, subproject_id FK SET NULL, provider, model, mode DEFAULT 'explore', status DEFAULT 'active', created_at, updated_at
- agent_messages: id TEXT PK, session_id FK CASCADE, role, content, tool_call JSON, created_at
- provenance_links: id TEXT PK, resource_id FK CASCADE, target_type, target_id, context, quote, confidence REAL DEFAULT 1.0, created_at
- git_configs: id TEXT PK, project_id FK CASCADE, remote_url, branch DEFAULT 'main', auto_push BOOLEAN DEFAULT FALSE, last_push_at, status DEFAULT 'disconnected'
- jobs: id TEXT PK, type, project_id FK, subproject_id FK, status DEFAULT 'queued', progress_pct INTEGER DEFAULT 0, progress_steps JSON, pid INTEGER, started_at, completed_at, error, result JSON, created_at
- search_index: FTS5 virtual table (entity_type, entity_id, project_id, title, content) with porter unicode61 tokenizer

All indexes as specified in the design.

### 4. backend/foundry/storage/models.py — Pydantic Models
- Define Pydantic models for each entity (for API serialization)
- Project, Subproject, Resource, Task, Note, etc.
- Use proper types (datetime, enums for status fields, etc.)

### 5. backend/foundry/events.py — Simple Event Bus
- In-process pub/sub for internal events
- async def emit(event_name, data)
- async def on(event_name, handler)
- Used for: activity feed updates, job progress, etc.

### 6. backend/foundry/main.py — FastAPI Application
- Create FastAPI app with lifespan (startup/shutdown)
- On startup: load config, init database, run migrations, log startup info
- Mount API router at /api
- Serve static frontend files from frontend/dist if it exists (production mode)
- CORS middleware for dev (allow localhost:5173)

### 7. backend/foundry/api/router.py — Main API Router
- Include sub-routers for each API module
- For now, only system.py needs real implementation

### 8. backend/foundry/api/system.py — System Endpoints
- GET /api/system/health — returns health status:
  ```json
  {
    "status": "healthy",
    "version": "0.1.0",
    "uptime_seconds": ...,
    "db": "ok" | "error",
    "workspace": "ok" | "error",
    "disk_free_gb": ...,
    "agent_provider": "ollama (connected)" | "none",
    "active_projects": 0,
    "active_ingestions": 0,
    "pending_reconcile_issues": 0
  }
  ```
- GET /api/system/config — returns non-secret config (masks API keys)
- GET /api/system/version — returns version info

### 9. backend/foundry/api/projects.py — Project Stubs
- GET /api/projects — list projects (empty for now, returns [])
- POST /api/projects — create project (stub, returns 501)
- GET /api/projects/{id} — get project (stub)
- These are stubs that will be filled in during MVP phase

### 10. backend/foundry/cli.py — CLI Entry Point
- Using Click
- `foundry start` — start uvicorn
- `foundry stop` — stop (by PID file)
- `foundry status` — check if running + call health endpoint
- `foundry health` — detailed health check
- `foundry version` — print version
- `foundry config show` — print current config (mask secrets)

### 11. backend/foundry/logging_setup.py — Structured Logging
- JSON lines format for log files
- Human-readable format for console
- Configurable level from config
- Create log directory if needed
- Rotated logs (10MB x 5)

### 12. backend/tests/conftest.py + basic tests

- Type hints on all functions
- Keep imports clean (ruff-compatible)
