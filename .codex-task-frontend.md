# Codex Task: Foundry Frontend Foundation

Build the frontend foundation for Foundry — a React + TypeScript + Vite application with dark theme, basic layout shell, routing, API client, and an empty-state dashboard.

## What Already Exists
- frontend/package.json with dependencies (React 19, TanStack Router/Query, Zustand, Tailwind 4, lucide-react, etc.)
- frontend/vite.config.ts with proxy to backend at :8121
- frontend/tsconfig.json with path aliases (@/*)
- frontend/index.html with dark body class
- Backend running on :8120 with /api/system/health endpoint

## What To Build

### 1. frontend/src/styles/globals.css
- Import Tailwind CSS v4 (use @import "tailwindcss")
- Define CSS custom properties for the dark theme:
  - Background: zinc-950
  - Surface: zinc-900
  - Border: zinc-800
  - Text: zinc-100
  - Text muted: zinc-400
  - Primary/accent: blue-500
  - Destructive: red-500
  - Success: emerald-500
- Keep it minimal — Tailwind utility classes do the heavy lifting

### 2. frontend/src/main.tsx
- React 19 createRoot
- Import globals.css
- Render App component

### 3. frontend/src/App.tsx
- Set up TanStack Router with routes
- Set up TanStack Query provider
- Routes:
  - / → Dashboard (index)
  - /project/:id → ProjectWorkspace (stub)
  - /onboarding → Onboarding (stub)
  - /settings → Settings (stub)

### 4. frontend/src/api/client.ts
- Typed API client using fetch (not axios)
- Base URL: "" (same origin, proxied in dev)
- Functions:
  - getHealth() → HealthResponse
  - getProjects() → Project[]
  - getConfig() → ConfigResponse
  - getVersion() → VersionResponse
- TypeScript interfaces for all responses matching the backend

### 5. frontend/src/stores/app.ts
- Zustand store for global app state:
  - sidebarOpen: boolean
  - agentPanelOpen: boolean
  - currentProjectId: string | null
  - agentMode: "explore" | "builder" | "full-override"

### 6. frontend/src/components/layout/Shell.tsx
- Main application shell with three areas:
  - Sidebar (left, collapsible)
  - Main content (center, fills remaining space)
  - Agent bar (bottom, thin bar)
- Use CSS grid or flexbox
- Dark theme: bg-zinc-950 text, zinc-900 surfaces, zinc-800 borders

### 7. frontend/src/components/layout/Sidebar.tsx

  - Foundry logo/name at top

### 8. frontend/src/components/layout/AgentBar.tsx

## Design Guidelines
- Dark theme throughout (zinc-950 bg, zinc-900 surfaces, zinc-800 borders)
- Clean, modern, minimal — not cluttered
- System font stack (no custom fonts for MVP)
- Consistent spacing: p-4 for panels, gap-2/gap-4 for layouts
- Rounded corners: rounded-lg on cards/panels
- Transitions: transition-colors on interactive elements
- Focus rings: ring-2 ring-blue-500/50 on focus-visible

## Rules
- TypeScript strict mode
- Functional components only
- Use Tailwind utility classes (no separate CSS files beyond globals.css)
- Keep components small and focused
- No placeholder "coming soon" text in components that won't be seen yet — only in stub route pages
- Use TanStack Router for navigation (NOT react-router)
- Use TanStack Query for data fetching (NOT useEffect+fetch)
